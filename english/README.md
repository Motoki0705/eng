# english ディレクトリ構造

このリポジトリの `english/` は学習素材を「本体テキスト → 過去問 → テスト → 解答」の流れで整理しています。全体の位置づけは次のとおりです。

- `english/past_exam_questions/` : 過去問の原文。年度ごと・問題番号ごとに分割して保存。
- `english/text/` : 基本文法・語法などの解説テキスト。学習の基盤。
- `english/trands/` : 周辺資料や補助的な素材。出題傾向と対策。
- `english/tests/` : 過去問・テキストをもとにした演習問題。
- `english/ans/` : `english/tests/` の解答と解説。基本的に同じ構造で対応。

学習の流れ（関係性）
- `english/past_exam_questions/` で出題形式を確認
- `english/text/` で知識を整理
- `english/trands/` で出題傾向を把握
- `english/tests/` で演習
- `english/ans/` で答え合わせと理解の定着

## 福岡大学入試問題の構成

### 各大問の形式（2024年度基準）

| 大問 | 形式 | 問題数 | 特徴 |
|------|------|--------|------|
| **〔Ⅰ〕** | 下線部和訳 | 1問 | 長文中の1文を日本語訳 |
| **〔Ⅱ〕** | 長文読解 | 10選択肢から4つ選択 | 内容一致問題 |
| **〔Ⅲ〕** | 文法・語法4択 | 8問 (a)～(h) | 空所補充形式 |
| **〔Ⅳ〕** | アクセント | 6問 (a)～(f) | 強勢位置の判定 |
| **〔Ⅴ〕** | 整序英作文 | 4問 A～D | 7語から6語を選んで並べ替え（1語不要） |

## tests/ans/の拡張計画

### 基本方針

#### 1. 問題作成の原則
- **過去問準拠**: `past_exam_questions/` の出題形式を厳密に踏襲
- **トレンド分析**: `trands/` の傾向分析を反映し、頻出分野を重点的に扱う
- **自作問題**: Next Stageの原文をそのまま使わず、参照ナンバーの知識を問う新規問題を作成
- **Next Stage対応**: 各問題に対応するNext StageのNo.を明記

#### 2. 問題作成の手順
1. `past_exam_questions/` の該当年度を読み込み、出題形式・難易度を把握
2. `trands/` の該当problemを読み込み、頻出分野・重要ポイントを確認
3. `text/next_stage/` から該当分野の知識を抽出
4. 過去問の形式・難易度に合わせた自作問題を作成
5. 解答には詳細な解説とNext StageのNo.を明記

#### 3. 命名規則とディレクトリ構造
```
tests/（または ans/）
├── problem_1/              # 〔Ⅰ〕下線部和訳
├── problem_2/              # 〔Ⅱ〕長文読解
├── problem_3/              # 〔Ⅲ〕文法・語法4択
│   └── {No.}_{分野名}/     # 例: 17_動詞の語法_515_641/
│       ├── reference.md    # 使用済みNext Stage No.の管理ファイル
│       └── pair_XXX/       # 例: pair_001/
│           ├── A.md        # 問題セットA（8問）
│           ├── B.md        # 問題セットB（8問、A.mdと同じNo.、異なる順序・英文）
│           ├── C.md        # 問題セットC（オプション、さらに問題を追加する場合）
│           └── D.md        # 問題セットD（オプション）
├── problem_4/              # 〔Ⅳ〕アクセント
└── problem_5/              # 〔Ⅴ〕整序英作文
    └── {分野名}/           # 例: verb_patterns/
        ├── reference.md    # 使用済みNext Stage No.の管理ファイル
        └── pair_XXX/
            ├── A.md        # 問題セットA（4問: A～D）
            └── B.md        # 問題セットB（4問、同じNo.、異なる順序・英文）
```

#### 4. pairの定義と拡張方針

**pairの意味**:
- **同じpair内（例: pair_001/）**:
  - 参照するNext StageのNo.は共通
  - A.md, B.md, C.md, D.md... は、同じNo.を使いながら出題順序と英文内容を変えたバリエーション
  - 例: pair_001/A.mdもB.mdもNo.515, 523, 530を使うが、問題文と選択肢は異なる

- **異なるpair（例: pair_002/）**:
  - 異なる参照No.を用いた新しい問題セット
  - 例: pair_001はNo.515～550を使用、pair_002はNo.551～590を使用

**拡張方針**:
1. **同じNo.でバリエーションを増やす場合**: 既存のpair内にC.md, D.md...を追加
2. **新しいNo.で問題を作る場合**: pair_002/, pair_003/...を新設

#### 5. reference.mdによるNo.管理

各分野のディレクトリ（例: `english/tests/problem_3/17_動詞の語法/`）に`reference.md`を作成し、どのNo.をすでに使用したかを記録します。

**reference.mdの構造例**:
```markdown
# 17_動詞の語法 - Next Stage参照No.管理

## pair_001
**使用済みNo.**: 515, 523, 530, 535, 540, 546, 551, 558

| No. | 文法事項 | 問題ファイル |
|-----|---------|-------------|
| 515 | mind doing | A.md (a), B.md (c) |
| 523 | decide to do | A.md (b), B.md (a) |
| 530 | enjoy doing | A.md (c), B.md (e) |
| 535 | stop doing/to do | A.md (d), B.md (b) |
| 540 | remember doing/to do | A.md (e), B.md (f) |

| 551 | enable A to do | A.md (g), B.md (g) |
| 558 | prevent A from doing | A.md (h), B.md (h) |

**使用済みNo.**: 560, 565, 570, 575, 580, 585, 590, 598
（詳細は同様に記載）

## 未使用No.
```

**重要なポイント**:

1. **過去問準拠**: 出題形式を厳密に踏襲し、実践的な演習が可能
2. **トレンド分析**: 頻出分野を重点的に扱い、効率的な学習を実現
3. **自作問題**: Next Stageの原文をそのまま使わず、知識を問う新規問題を作成
4. **体系的管理**: reference.mdによるNo.管理で、重複や漏れを防止
5. **段階的拡張**: pairとA/B/C/Dファイルの組み合わせで、柔軟な問題追加が可能

### 問題・解答フォーマットの統一

- **tests/**: 純粋に問題のみ（解答・解説なし）
- **ans/**: 詳細な解答・解説、Next Stage対応表を完備
- **reference.md**: 使用済みNo.の管理と追跡

### 次のステップ

フェーズ1から順次実装を進め、最終的には全5大問×複数pairの包括的な演習問題セットを構築します。各問題セットは過去問分析とNext Stageの知識を組み合わせた自作問題により、実践的かつ体系的な学習を可能にします。
