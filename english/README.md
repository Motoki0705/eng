## ディレクトリ構造

このリポジトリの `english/` は学習素材を「本体テキスト → 過去問 → テスト → 解答」の流れで整理しています。

- `english/past_exam_questions/` : 過去問の原文。年度ごと・問題番号ごとに分割して保存。
- `english/text/` : 基本文法・語法などの解説テキスト。学習の基盤。
- `english/trands/` : 周辺資料や補助的な素材。出題傾向と対策。
- `english/tests/` : 過去問・テキストをもとにした演習問題。
- `english/ans/` : `english/tests/` の解答と解説。基本的に同じ構造で対応。各pairには`knowledge.md`（習得すべき知識）を配置。

**学習の流れ（関係性）**:
1. `english/past_exam_questions/` で出題形式を確認
2. `english/text/` で知識を整理
3. `english/trands/` で出題傾向を把握
4. `english/ans/{problem}/pair_XXX/knowledge.md` で事前知識を習得
5. `english/tests/` で演習
6. `english/ans/` で答え合わせと理解の定着

## 福岡大学入試問題の構成

### 各大問の形式（2024年度基準）

| 大問 | 形式 | 問題数 | 特徴 |
|------|------|--------|------|
| **〔Ⅰ〕** | 下線部和訳 | 1問 | 長文中の1文を日本語訳 |
| **〔Ⅱ〕** | 長文読解 | 10選択肢から4つ選択 | 内容一致問題 |
| **〔Ⅲ〕** | 文法・語法4択 | 8問 (a)～(h) | 空所補充形式 |
| **〔Ⅳ〕** | アクセント | 6問 (a)～(f) | 強勢位置の判定 |
| **〔Ⅴ〕** | 整序英作文 | 4問 A～D | 7語から6語を選んで並べ替え（1語不要） |

## 基本方針とルール

このセクションでは、問題作成における基本的な方針、ディレクトリ構造、命名規則を定義します。

### 問題作成の基本方針

#### 1. 問題作成の原則
- **過去問準拠**: `past_exam_questions/` の出題形式を厳密に踏襲
- **トレンド分析**: `trands/` の傾向分析を反映し、頻出分野を重点的に扱う
- **自作問題**: Next Stageの原文をそのまま使わず、参照ナンバーの知識を問う新規問題を作成
- **Next Stage対応**: 各問題に対応するNext StageのNo.を明記

#### 2. 問題作成の手順
1. `past_exam_questions/` の該当年度を読み込み、出題形式・難易度を把握
2. `trands/` の該当problemを読み込み、頻出分野・重要ポイントを確認
3. `text/next_stage/` から該当分野の知識を抽出
4. 過去問の形式・難易度に合わせた自作問題を作成
5. 解答には詳細な解説とNext StageのNo.を明記

#### 3. 命名規則とディレクトリ構造
```
tests/
├── problem_1/              # 〔Ⅰ〕下線部和訳
├── problem_2/              # 〔Ⅱ〕長文読解
├── problem_3/              # 〔Ⅲ〕文法・語法4択
│   └── {No.}_{分野名}/     # 例: 17_動詞の語法_515_641/
│       ├── reference.md    # 使用済みNext Stage No.の管理ファイル
│       └── pair_XXX/       # 例: pair_001/
│           ├── A.md        # 問題セットA（8問）
│           ├── B.md        # 問題セットB（8問、A.mdと同じNo.、異なる順序・英文）
│           ├── C.md        # 問題セットC（オプション）
│           └── D.md        # 問題セットD（オプション）
├── problem_4/              # 〔Ⅳ〕アクセント
└── problem_5/              # 〔Ⅴ〕整序英作文
    └── {分野名}/
        ├── reference.md
        └── pair_XXX/
            ├── A.md
            └── B.md

ans/
├── problem_1/
├── problem_2/
├── problem_3/
│   └── {No.}_{分野名}/
│       └── pair_XXX/
│           ├── knowledge.md  # 📚 習得すべき知識（最初に読む）
│           ├── A.md          # A.mdの解答と詳細解説
│           └── B.md          # B.mdの解答と詳細解説
├── problem_4/
└── problem_5/
    └── {分野名}/
        └── pair_XXX/
            ├── knowledge.md  # 📚 習得すべき知識（最初に読む）
            ├── A.md
            └── B.md
```

#### 4. pairの定義と拡張方針

**pairの意味**:
- **同じpair内（例: pair_001/）**:
  - 参照するNext StageのNo.は共通
  - A.md, B.md, C.md, D.md... は、同じNo.を使いながら出題順序と英文内容を変えたバリエーション
  - 例: pair_001/A.mdもB.mdもNo.515, 523, 530を使うが、問題文と選択肢は異なる

- **異なるpair（例: pair_002/）**:
  - 異なる参照No.を用いた新しい問題セット
  - 例: pair_001はNo.515～550を使用、pair_002はNo.551～590を使用

**拡張方針**:
1. **同じNo.でバリエーションを増やす場合**: 既存のpair内にC.md, D.md...を追加
2. **新しいNo.で問題を作る場合**: pair_002/, pair_003/...を新設

#### 5. reference.mdによるNo.管理

各分野のディレクトリ（例: `english/tests/problem_3/17_動詞の語法/`）に`reference.md`を作成し、どのNo.をすでに使用したかを記録します。

#### 6. knowledge.mdの役割

各pairの`ans/`ディレクトリに`knowledge.md`を配置します。このファイルは**問題を解く前に読むべき事前知識**をまとめたもので、Next Stageの該当ナンバーの要点を効率的に学習できるようにしています。

**knowledge.mdの目的**:
- 問題を解く前に必要な文法知識を確認
- Next Stageの該当ナンバーの要点を凝縮
- 学習効率の向上（予習 → 問題 → 答え合わせ → 復習の流れ）

**配置場所**: `ans/{problem_N}/{分野名}/pair_XXX/knowledge.md`

#### 7. 学習フローとナビゲーションリンク

各pairでの学習は以下の順序で行います。ファイル間にナビゲーションリンクを設置し、スムーズな学習フローを実現します。

**学習フロー（pair内）**:
```
📚 knowledge.md（事前知識）
      ↓
📝 tests/A.md（問題A）
      ↓
✅ ans/A.md（解答A）
      ↓
📝 tests/B.md（問題B）
      ↓
✅ ans/B.md（解答B）
      ↓
（C.md, D.md... があれば続く）
```

**ナビゲーションリンクの形式**:

各ファイルの先頭または末尾に以下のようなリンクを記載します：

```markdown
---
## 🔗 ナビゲーション

| 前のステップ | 現在 | 次のステップ |
|-------------|------|-------------|
| ← [knowledge.md](../../../../ans/problem_3/17_動詞の語法_515_641/pair_001/knowledge.md) | 📝 A.md（問題） | [解答を見る →](../../../../ans/problem_3/17_動詞の語法_515_641/pair_001/A.md) |
```

**各ファイルのリンク構成**:

| ファイル | 前のリンク | 次のリンク |
|---------|-----------|-----------|
| knowledge.md | （なし） | tests/A.md |
| tests/A.md | knowledge.md | ans/A.md |
| ans/A.md | tests/A.md | tests/B.md |
| tests/B.md | ans/A.md | ans/B.md |
| ans/B.md | tests/B.md | knowledge.md（復習）または次のpair |
